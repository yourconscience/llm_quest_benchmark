How it works

Give a language model a text quest: read the situation, pick an action, repeat for 10-50 turns. Run the same quest with five different agent architectures - from a bare prompt to a planning agent with tools. Compare success rates, exploration patterns, and cost.

The quests come from Space Rangers: community-authored text adventures with branching narratives and non-obvious optimal paths. They were not built for LLM evaluation, which makes the difficulty organic rather than synthetic.

Agent modes

A Baseline

Minimal prompt. Pick a number. No reasoning.

B Reasoning

Structured chain-of-thought before choosing.

C Knowledge

Reasoning plus domain hints about game mechanics.

D Planner

Multi-turn planning with strategy updates.

E Tools

State tracker, calculator, quest history.

Metrics

  • Success rate - fraction of runs ending in quest success
  • Exploration rate - unique locations visited / total available
  • Repetition rate - how often the agent repeats recent actions (>0.6 predicts 0% success)
  • Cost per run - total token cost in USD

Open source

Everything is on GitHub: benchmark framework, evaluation data, configs. Supports Docker and local dev with uv. Adding a new model requires only a YAML config change.

The quest engine is space-rangers-quest by roginvs.
