Does the wrapper around an LLM matter more than the model itself?

Kirill Korikov · May 2026

Most LLM benchmarks hold the agent architecture constant and vary the model. We did the opposite: same models, five different agent wrappers, dozens of text quests. The results surprised us.

The setup

Give an LLM a text adventure: read the situation, pick from a list of actions, repeat for 10-50 turns until you win or lose. Now do the same thing five different ways:

  • Baseline (A) - bare prompt, just pick a number
  • Reasoning (B) - structured chain-of-thought before choosing
  • Knowledge (C) - reasoning plus domain hints about game mechanics
  • Planner (D) - multi-turn planning with strategy updates
  • Tools (E) - reasoning plus state tracker, calculator, quest history

The quests come from Space Rangers, a corpus of community-authored text adventures with branching narratives and non-obvious optimal paths. They were not designed for LLM evaluation, which is part of why they work: the difficulty is organic, not synthetic.

We tested six mid-tier production models - the tier developers actually deploy, not frontier flagships. All in the ELO 1400-1475 range, $0.01-0.04 per run.

What we found

Prompted reasoning is the single biggest unlock

Moving from bare prompting (A) to structured reasoning (B) produced the largest jump in success rate across all models. Simply asking the model to analyze before choosing was worth more than switching to a more expensive model.

Models get stuck in loops - and it predicts failure

A repetition rate above 0.6 (repeating recent actions more than 60% of the time) predicted 0% success rate across every model we tested. Loop detection could serve as a cheap early stopping signal.

Different architectures help different problem types

Spatial puzzles need tools (state tracking). Long quests need planning (to avoid loops). Domain-scored quests need knowledge injection. No single architecture dominates across all quest types.

Cost-performance tradeoffs are real

DeepSeek V3.2 at $0.01/run with good scaffolding outperformed more expensive models with minimal scaffolding. The cheapest model is not always the worst choice if you invest in the wrapper.

The practical takeaway: if you are building an LLM agent, spend time on the architecture before upgrading the model. A structured prompt template costs nothing and often delivers more than a tier jump in model capability.

The failure taxonomy

Analyzing 261 runs across the hardest quests revealed four distinct failure categories, each addressable by a different architectural intervention:

Spatial confusionModel cannot track coordinates or grid stateState tracker tool (E)

Missing domain knowledgeModel lacks game-specific scoring rulesKnowledge hints (C)
